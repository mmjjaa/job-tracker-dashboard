# PRD — job-notion-clipper

**버전**: v1.0 MVP  
**작성일**: 2026-05-17  
**대상 독자**: React 기초 수준 개발자 (학습 + 실용 목적)

---

## 1. 문제 정의

채용 활동 중 공고를 발견하면 일일이 복사·붙여넣기로 스프레드시트나 Notion에 옮겨야 한다. 이 과정에서:

- 회사명·포지션·기술스택을 수동으로 입력해야 함
- 마감일을 놓치는 경우 발생
- 어떤 공고를 봤는지 기억이 안 남

---

## 2. 목표

채용공고 페이지에서 아이콘 한 번 클릭으로 Notion 지원 DB에 구조화된 데이터를 자동 저장한다.

---

## 3. 사용자

- React 기초 학습자 / 주니어 개발자
- 적극적으로 채용 활동 중인 사람
- Notion을 일정·업무 관리 도구로 쓰는 사람

---

## 4. 기능 요구사항

### 4-1. 핵심 기능 (MVP 필수)

#### F-01. 공고 텍스트 추출
- 원티드 / 링크드인 / 잡코리아 / 사람인 4개 사이트에서 공고 본문 추출
- 사이트별 CSS 선택자로 대상 영역 지정, 실패 시 `body.innerText` 폴백
- content script → background로 텍스트 전달

#### F-02. AI 구조화 파싱
- Claude API(`claude-haiku-4-5`)로 다음 필드 추출
  - 회사명, 포지션, 기술스택(배열), 연봉(없으면 "정보 없음"), 마감일(ISO 8601 또는 null)
- JSON 형식으로 응답 강제 (`response_format` 프롬프트)
- 파싱 실패 시 빈 값으로 폴백, 에러 팝업 표시

#### F-03. Notion DB 저장
- Notion API로 지정 DB에 페이지 생성
- 저장 스키마: 회사명(title) / 포지션 / URL / 기술스택 / 연봉 / 지원현황(기본값: "관심") / 마감일 / 한줄메모

#### F-04. 팝업 미리보기
- AI 추출 결과를 팝업에서 확인 후 저장 여부 결정
- 로딩 스피너 (파싱 중) → 미리보기 카드 (완료)

---

### 4-2. 부가 기능 (MVP 포함)

#### F-05. 메모 입력 (①)
- 저장 전 `textarea`에 한 줄 메모 입력 가능
- 메모는 Notion `한줄메모` 필드에 함께 저장
- 빈 값이면 AI 생성 요약으로 채움

#### F-06. 중복 감지 (②)
- 저장 버튼 클릭 전, 현재 URL을 `chrome.storage.local`에 저장된 URL 목록과 비교
- 이미 저장된 공고라면 "이미 저장된 공고예요 ✓" 배너 표시
- 덮어쓰기 또는 취소 선택 제공

#### F-07. 최근 저장 목록 탭 (③)
- 팝업 상단에 탭 2개: [저장하기] / [최근 목록]
- 최근 저장 5개를 `chrome.storage.local`에 캐싱
- 각 항목: 회사명 + 포지션 + 마감일 + 지원현황 표시
- 항목 클릭 시 해당 Notion 페이지 새 탭으로 열기

## 7. 데이터 흐름

```
[공고 페이지]
    │
    ▼  content.ts: DOM 선택자로 텍스트 추출
[background.ts]
    │
    ├── claude.ts: Claude API 호출 → JSON 파싱
    │
    ├── notion.ts: Notion API로 페이지 생성
    │
    └── chrome.storage.local: URL + 요약 5개 캐싱
    │
    ▼
[Popup.tsx]
    ├── SaveTab.tsx: 미리보기 + 메모 입력 + 저장 버튼
    └── RecentTab.tsx: 최근 5개 목록
```
